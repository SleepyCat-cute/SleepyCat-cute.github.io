## 1. この文書の使い方

この文書は、「個人LINEでのやり取りをやめたいが、何をどれだけ費用をかければよいか分からない」という状況に対して、業界の標準的な選択肢と費用感を整理したものです。経営層への説明では、4章の比較表と5章の3段階プランをそのまま提示資料として使えます。金額はすべて2026年7月時点の情報です。

---

## 2. なぜ個人LINEではだめか(公的根拠3点)

現場感覚での「便利だから」ではなく、公的なガイドラインと個人LINEの技術的な仕組みの両面から、業務利用を避けるべき理由は明確です。

**① 厚労省ガイドラインが「管理されたもの」しか認めていない**
厚生労働省「医療情報システムの安全管理に関するガイドライン」第7.0版(令和8年6月改定)は、私物端末の業務利用(BYOD)について「原則禁止ではなく、十分な安全性確保を条件に許容」という位置づけを取っています。ただし条文は「管理されていない端末でのBYODは許可してはならない」と明記しており、個人LINEアプリが入った私物スマホをそのまま業務利用することは、この「管理されていない」状態に該当します。またクラウドサービス全般について「医療機関等が定める安全管理の基準やルールに適合したサービスを利用させること」「許可されていないサービスの利用禁止を規則等に盛り込む」ことを求めており、個人LINEのような汎用SNSを組織として管理・許可した形跡がない状態での利用は、この一般原則に抵触します。✅(出典: 厚労省第7.0版通知 https://www.mhlw.go.jp/stf/shingi/0000516275_00006.html )

**② 個人LINEは管理者がログも削除も制御できない**
LINE WORKS等の業務用ツールには、管理者が操作履歴(誰が・いつ・どこから接続したか)を監査する機能や、端末紛失時にアカウントを遠隔で停止する機能があります。個人LINEにはこうした組織管理・監査・強制ログアウトの仕組みが存在しません。誰が利用者情報をいつ誰に送ったか、事業所側は把握も統制もできず、職員が退職してもその職員のスマホに残ったトーク履歴を事業所側で消すこともできません。✅(出典: LINE WORKS管理者ガイド https://help.worksmobile.com/ja/admin-guides/audit/ )

**③ 患部写真は私物端末とLINE社のクラウドに自動で残る**
個人LINEで撮影・送信した写真は、送信者・受信者双方の私物スマホ内(カメラロールやLINEのキャッシュ)に自動保存される仕組みになっています。電子カルテのように「システム内だけで完結し端末には残らない」設計にはなっておらず、スマホの紛失・盗難・廃棄・修理業者への引き渡し時に、患部写真や氏名がそのまま外部に流出するリスクを常に抱えています。これは技術的な構造の問題であり、職員個人の注意力では防ぎきれません。

## 2-1. なぜ共有アカウントでもだめか(MCSも例外ではない)

「個人LINEをやめてMCSに変えればそれで解決」ではありません。**このステーションはすでにMCSを導入済みですが、正社員1名(管理者)の1アカウントを全員で共用し、社用メールがないため個人のフリーメール(yahoo.com)で登録・共用しています。**1つのアカウントを複数職員で共用する運用は、個人LINEとは別の理由で問題になります。

厚労省ガイドライン第7.0版 第14章(認証・認可に関する安全管理措置)は、ID・パスワード等の識別・認証情報を「**本人しか知り得ない状態**」に保つことを義務付け、「**全ての職員及び関係者**」に対し本人の識別・認証手段を用意するよう求めています。この要件は「**小規模な医療機関等で利用者が限定される場合においても、同様**」と明記されており、7名規模だから対象外という理屈は成り立ちません。✅(出典: 厚労省第7.0版システム運用編PDF 第14章・14.1.1項)

共用アカウントの問題点:
- 誰がいつ何を投稿・閲覧したか個人単位で追跡できない(IDの台帳管理・棚卸しも実質不可能)
- 退職者が出てもパスワードを変えない限り元職員がアクセスし続けられる
- 「本人しか知り得ない状態」という要件に正面から反する

→ 是正の方向性は「MCSをやめる」ことではなく、「**今の共用アカウントを人数分の個別アカウントに分ける**」ことです。詳細は4章プラン0・6章導入ステップ案を参照してください。

## 4. 7名規模の3段階プラン(月額)

### プラン0: 0円 — MCSの適正化(個別アカウント化)+運用ルール+誓約書

**新規導入ではありません。**すでにあるMCSフリープランを、共用の1アカウントから職員1人1アカウントに整え直すだけです。

| 項目 | 内容 | 月額 |
|---|---|---|
| コミュニケーション基盤 | 既存のMCSフリープランを個別アカウント化(職員1人につき1アカウント、患者登録100件・グループ30件まで) | 0円 |
| メールアドレス | 共用のフリーメール(yahoo.com等)での登録をやめ、職員各自のメールアドレスで再登録 | 0円 |
| 規程・誓約書 | 公開ひな形を叩き台に自作 | 0円 |
| 運用ルール | 「患者情報はMCSでのみ・個人LINE業務利用禁止・アカウント共用禁止」を明文化 | 0円 |
| **合計** | | **0円** |

**防げるリスク**: 患部写真・氏名・病名を非公開型・招待制の環境でやり取りできるようになり、誤送信や退職者のトーク履歴残存等、個人LINE特有のリスクを大幅に低減。**加えて、個別アカウント化により「誰がいつ何を見たか」の追跡が可能になり、ガイドライン第14章の識別・認証要件(2-1章参照)を満たす状態に是正できる。**
**防げないリスク**: 組織的なログ監査、端末紛失時の遠隔データ消去(MCS自体にはこの機能がない)。

### プラン1: 約3,000円台 — LINE WORKS等

| 項目 | 内容 | 月額(7名) |
|---|---|---|
| コミュニケーション基盤 | LINE WORKSスタンダード(年額契約) | 450円×7名=3,150円 |
| 規程・誓約書 | プラン0と同様 | 0円 |
| **合計** | | **約3,150円** |

**防げるリスク**: プラン0の効果に加え、誰が・いつ・どこから接続したかの操作履歴監査、端末紛失・盗難時のアカウント遠隔停止が可能に。業務用アカウントと私用アカウントを制度的に分離できる。
**防げないリスク**: 保険請求管理機能はなし。写真データはLINE WORKS上のストレージに残るため、電子カルテほど「端末に一切残さない」設計ではない。

### プラン2: 約3.3万円 — カイポケ+端末7台+MDM

| 項目 | 内容 | 月額(7名) | 確認状態 |
|---|---|---|---|
| 訪問看護専用電子カルテ | カイポケ訪問看護(1拠点定額) | 25,000円(税別) | ✅ |
| 業務用端末(格安SIM回線) | 7台(1台あたり月額1,000円前後) | 約7,000円 | ⚠️目安 |
| MDM(端末の遠隔管理) | 7台(1台あたり月額200〜300円) | 約1,400〜2,100円 | ⚠️目安 |
| **合計** | | **約33,400〜34,100円** | |

**防げるリスク**: 写真・記録がシステム内で完結し端末に一切残らないため、端末紛失・盗難時の情報漏洩リスクを構造的に排除。保険請求管理も一元化され、請求ミス防止・業務効率化も同時に実現。MDMで会社が端末を一元管理し、退職時のアカウント削除・データ消去も確実に実行できる。
**防げないリスク・限界**: 3プラン中もっとも投資額が大きく、経営層への説明では請求効率化等の副次効果も併せて訴求する必要がある。

### 写真運用ルール(そのまま掲示・回覧できる形)

1. 患部・利用者の写真撮影は、業務用端末またはMCS/電子カルテのアプリ内カメラ機能のみで行う。
2. 私物端末での撮影・保存を禁止する。
3. 撮影した写真は業務用端末のカメラロールに残さず、システムへのアップロード後は端末側のデータを削除する(電子カルテの多くはこの手順が不要な設計)。
4. 個人LINE・個人メール・私物クラウドストレージへの転送を禁止する。

## 6. 導入ステップ案(いきなり全部ではなく小さく始める)

**すでにMCSはあるため、「導入」ではなく「個別アカウント化」からのスタートです。**

| 段階 | 期間 | 内容 |
|---|---|---|
| ステップ1: 個別アカウントの試行発行 | 2週間 | 既存の共用アカウントは残したまま、希望者2〜3名分の個別アカウントをMCSで新規発行し、各自のメールアドレスでの運用感を確認。連携クリニックがすでに全員分のアカウント作成を依頼している事実を後押しに使う |
| ステップ2: 全員の個別アカウント化 | 試行後 | 問題なければ残り全職員も個別アカウントに切り替え、運用ルール・誓約書を正式配布 |
| ステップ3: 共用アカウントの廃止・個人LINE禁止の明文化 | 全員展開後 | 元の共用アカウント(共用フリーメール)を廃止し、個人情報保護規程・BYODポリシーに「個人LINEでの業務利用禁止・アカウント共用禁止」を明記し、周知徹底 |

一度にすべてを変えようとすると現場の反発が大きくなるため、まず0円のプラン0(個別アカウント化)から小さく始め、必要に応じてプラン1・2への移行を検討する順序が現実的です。

## 7. よくある反対と答え

| 反対意見 | 答え |
|---|---|
| 「金がない」 | プラン0(既存MCSの個別アカウント化+運用ルール+誓約書)は月額0円で是正できる。新規契約は不要。 |
| 「もうMCSは使ってるじゃないか」 | 共用の1アカウントを全員で使い回している状態は、厚労省ガイドライン第14章が求める「本人しか知り得ない状態」を満たしていない(2-1章参照)。ツールは合っているが運用が未完成、という位置づけ。連携クリニックからも個別アカウント化をすでに依頼されている。 |
| 「使い方を覚えられない」 | 個別アカウントもMCSそのものの操作感は変わらない。LINE WORKSへの移行を検討する場合も個人LINEに近い操作感で、2週間の試行で実際に触ってから判断できる。 |
| 「今まで問題なかった」 | 「今まで問題が表面化していない」ことと「リスクがない」ことは別。詳細は「01」資料を参照。 |
